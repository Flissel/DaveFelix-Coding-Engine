# 🚀 Coding Engine Container

Autonome Code-Generierung im Docker-Container mit Live-Preview und Web-Widget.

## Features

- 🐳 **Docker containerized** - Ubuntu 22.04 + Python 3.11 + Node.js 20
- 🖥️ **Live VNC preview** - Watch code execute in real-time via browser
- 🎛️ **REST API** - Full control via FastAPI endpoints
- 📡 **WebSocket streaming** - Real-time events and logs
- 🔌 **Embeddable widget** - Drop into any website
- ⏹️ **User-controlled stop** - No automatic timeouts
- 🔍 **Auto project detection** - Electron, Web Apps, CLI tools
- 📦 **Git Integration** - Automatic GitHub repository creation and push

## Quick Start

```bash
# Navigate to infra directory
cd infra

# Start the container
docker-compose up --build

# Access the services:
# - Control API: http://localhost:8000
# - Widget:      http://localhost:8000/widget/widget.html
# - VNC:         http://localhost:6080/vnc.html
```

## Git Integration

The container supports automatic GitHub repository creation and code pushing when generation completes.

### Setup

1. **Create a GitHub Personal Access Token (PAT)**
   - Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Tokens (classic)
   - Generate new token with `repo` scope
   - Copy the token

2. **Set the environment variable**
   
   **Option A: Environment variable (recommended)**
   ```bash
   # Linux/macOS
   export GITHUB_TOKEN=ghp_your_token_here
   cd infra && docker-compose up
   
   # Windows PowerShell
   $env:GITHUB_TOKEN="ghp_your_token_here"
   cd infra; docker-compose up
   
   # Windows CMD
   set GITHUB_TOKEN=ghp_your_token_here
   cd infra && docker-compose up
   ```
   
   **Option B: .env file**
   ```bash
   # Create infra/.env file
   echo "GITHUB_TOKEN=ghp_your_token_here" > infra/.env
   ```

### Usage via API

**Check Git status:**
```bash
curl http://localhost:8000/api/git/status
```

Response:
```json
{
  "configured": true,
  "username": "your-username",
  "message": "GitHub configured for user: your-username"
}
```

**Start engine with Git enabled:**
```bash
curl -X POST http://localhost:8000/api/start \
  -H "Content-Type: application/json" \
  -d '{
    "requirements_file": "my-project.json",
    "output_dir": "my-output",
    "git_config": {
      "repo_name": "my-generated-app",
      "description": "Auto-generated by Coding Engine",
      "private": true,
      "create_repo": true,
      "push_on_complete": true
    }
  }'
```

**Manual push (after generation):**
```bash
curl -X POST "http://localhost:8000/api/git/push?output_dir=my-output" \
  -H "Content-Type: application/json" \
  -d '{
    "repo_name": "my-repo",
    "private": true
  }'
```

### Usage via Widget

1. Open the widget: `http://localhost:8000/widget/widget.html`
2. Go to the **Config** tab
3. Enable **"Push to GitHub on completion"**
4. Optionally set repository name (defaults to output directory name)
5. Choose private/public
6. Click **Start**
7. When complete, a banner shows the repository URL

### Git Events (WebSocket)

The WebSocket stream includes these Git-related events:

| Event | Description |
|-------|-------------|
| `git_repo_created` | Repository created on GitHub |
| `git_push_started` | Push operation started |
| `git_push_complete` | Code successfully pushed (includes `repo_url`) |
| `git_error` | Git operation failed |

## Development

### Live Editing

The docker-compose.yml mounts `control_server/` and `widget/` directories, allowing live editing without rebuild:

```bash
# Edit files locally - changes reflect immediately
vim infra/widget/widget.js

# Only need to rebuild for Dockerfile changes
docker-compose up --build
```

### Adding New Features

1. **New API endpoint**: Edit `control_server/server.py`
2. **New data models**: Edit `control_server/models.py`
3. **Widget UI**: Edit `widget/widget.html` and `widget.css`
4. **Widget logic**: Edit `widget/widget.js`

## Troubleshooting

### Container won't start
```bash
# Check Docker is running
docker info

# Remove old containers and rebuild
docker-compose down
docker-compose up --build
```

### VNC not connecting
```bash
# Check Xvfb is running in container
docker-compose exec coding-engine ps aux | grep Xvfb

# Check x11vnc
docker-compose exec coding-engine ps aux | grep x11vnc
```

### Git push failing
```bash
# Check token is set
curl http://localhost:8000/api/git/status

# Verify token permissions
# Token needs 'repo' scope for private repos
# Token needs 'public_repo' scope for public repos only
```

### API not responding
```bash
# Check container health
docker-compose ps

# View logs
docker-compose logs -f coding-engine
```
